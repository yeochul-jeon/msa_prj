## 2. 상품 상태 이벤트 vs 데이터 변경 구분

### 2.1 핵심 판단 기준

```
┌─────────────────────────────────────────────────────────────┐
│                    핵심 판단 기준                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  "이 변화로 다른 팀이나 외부 시스템이 반응해야 하는가?"    │
│                                                             │
│  예 → 도메인 이벤트 가능성 높음 🟧                         │
│  아니오 → 단순 데이터 변경 (제외)                          │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  추가 판단 기준:                                           │
│  • 상품 상태(판매중/중지/품절)가 변경되는가?               │
│  • 검색팀이 재색인해야 하는가?                             │
│  • 주문팀이 주문 가능 여부를 변경해야 하는가?              │
│  • 판매자에게 알림이 가는가?                               │
│  • 가격/재고가 변경되어 다른 팀이 알아야 하는가?           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 판단 체크리스트 (3단계)

이벤트 후보가 도메인 이벤트인지 판단할 때 아래 질문을 순서대로 확인하세요.

```
□ 1. 상품의 비즈니스 상태가 변경되는가?
     → 아니오: 단순 데이터 변경 가능성 높음
     → 예: 다음 질문으로

□ 2. 다른 팀/시스템이 이 변화에 반응해야 하는가?
     (검색, 주문, 프로모션, 판매자 등)
     → 아니오: 단순 데이터 변경 (제외)
     → 예: 다음 질문으로

□ 3. 이 변화의 결과가 즉시 반영되어야 하는가?
     → 아니오: 🩷 핫스팟으로 표시 (배치 vs 실시간 논의)
     → 예: 도메인 이벤트! 🟧
```

```

│  ├──────────────────────────────────────────────────────┤  │
│  │  상품 등록 요청됨          │  상품 등록 승인됨       │  │
│  │  상품 수정 요청됨          │  상품 수정 승인됨       │  │
│  │  가격 변경 요청됨          │  가격 변경 승인됨       │  │
│  │  판매 중지 요청됨          │  판매 중지 처리됨       │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  교정 방법:                                                │
│  "이 이벤트는 판매자가 버튼을 눌러서 발생한 건가요,       │
│   아니면 플랫폼/관리자가 처리한 결과인가요?                │
│   판매자 행동 → '요청' 이벤트                              │
│   플랫폼 처리 → '승인/반려' 이벤트"                        │
│                                                             │
│  예시:                                                     │
│  • 판매자: "가격 변경 요청" 버튼 클릭                      │
│    → 🟧 "가격 변경이 요청되었다"                          │
│  • 시스템/관리자: 검토 후 승인                             │
│    → 🟧 "가격 변경이 승인되었다"                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 실수 7: API 호출을 이벤트로

```
┌─────────────────────────────────────────────────────────────┐
│              실수 7: API 호출을 이벤트로 표현               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ❌ 나쁜 예:                                               │
│  • "색인 API가 호출되었다"                                 │
│  • "검색 엔진에 요청이 전송되었다"                         │
│  • "물류 시스템 API가 호출되었다"                          │
│  • "외부 시스템 연동이 실행되었다"                         │
│                                                             │
│  ✅ 좋은 예:                                               │
│  • "색인이 요청되었다" → 커맨드 트리거                     │
│  • "색인이 완료되었다" → 결과 이벤트 🟧                   │
│  • "색인이 실패했다" → 결과 이벤트 🟧                     │
│                                                             │
│  원칙:                                                     │
│  ┌────────────────────────────────────────────────────┐    │
│  │  API 호출 = 커맨드 (🟦)의 실행                     │    │
│  │  요청 완료 = 상태 변경 = 이벤트 (🟧)               │    │
│  │  응답 수신 = 결과 이벤트 (🟧)                      │    │
│  └────────────────────────────────────────────────────┘    │
│                                                             │
│  흐름 예시:                                                │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  🟦 색인 요청하기  →  🟧 색인이 요청되었다           │  │
│  │  (커맨드)              (요청 이벤트)                 │  │
│  │                             │                        │  │
│  │                             ▼                        │  │
│  │                        🟩 검색 엔진                  │  │
│  │                             │                        │  │
│  │                             ▼                        │  │
│  │                        🟧 색인이 완료되었다          │  │
│  │                        (결과 이벤트)                 │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  교정 방법:                                                │
│  "'API 호출'을 '요청'으로, '응답'을 '완료/실패'로         │
│   바꿔 표현해보세요."                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 실수 8: 상품 상태 변경 이벤트 누락

```
┌─────────────────────────────────────────────────────────────┐

│                                                             │

│  ┌──────────────────────────────────────────────────────┐  │
│  │           상품 상태 전이 다이어그램                   │  │
│  │                                                       │  │
│  │  🟧 등록됨 → 🟧 검수 승인됨 → 🟧 판매 시작됨        │  │
│  │                                    │                 │  │
│  │                      ┌─────────────┼─────────────┐   │  │
│  │                      │             │             │   │  │
│  │                      ▼             ▼             ▼   │  │
│  │                 🟧 품절됨   🟧 판매중지됨   🟧 단종됨│  │
│  │                      │             │                 │  │
│  │                      └──────┬──────┘                 │  │
│  │                             ▼                        │  │
│  │                      🟧 판매 재개됨                  │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  필수 이벤트 체크리스트:                                   │
│  □ 상품이 등록되었다                                       │
│  □ 검수가 승인/반려되었다                                  │
│  □ 상품 판매가 시작되었다                                  │
│  □ 상품이 품절되었다                                       │
│  □ 상품 판매가 중지되었다                                  │
│  □ 상품 판매가 재개되었다                                  │
│  □ 상품이 단종되었다                                       │
│                                                             │
│  교정 방법:                                                │
│  "등록 후에 이 상품이 어떤 상태들을 거치나요?              │
│   판매 중지, 품절, 단종 같은 상태도 있지 않나요?           │
│   각 상태 변경마다 이벤트가 필요합니다."                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 자가 점검 체크리스트

```
작성한 이벤트 점검하기:

□ 과거형으로 작성했는가? ("~되었다")
□ 비즈니스 상태 변경이 발생하는 지점인가?
□ 다른 팀이 이 변화에 반응해야 하는가?
□ 단순 데이터 변경이 아닌 결과인가?
□ API 호출이 아닌 비즈니스 상태로 표현했는가?
□ 가격 종류(정가/판매가/할인가)가 명확한가?
□ 판매자 행동과 플랫폼 처리를 구분했는가?
□ 재고 이벤트 소유권이 명확한가?
□ 검수 단계를 과도하게 세분화하지 않았는가?
□ 상품 상태 전이(시작/중지/품절/단종)를 모두 표현했는가?
```
