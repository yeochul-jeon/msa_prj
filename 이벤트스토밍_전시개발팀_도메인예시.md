# 이벤트 스토밍 전시 개발팀 도메인별 예시

> 이 문서는 [이벤트스토밍_전시개발팀_가이드.md](./이벤트스토밍_전시개발팀_가이드.md)의 보충 자료입니다.

---

## 1. 개요

### 1.1 사용 방법

이 문서는 전시 개발팀 이벤트 스토밍 워크샵에서 **참조용 예시 자료**로 활용합니다.

- 도메인별로 어떤 이벤트, 커맨드, 애그리게이트가 있는지 참고
- 다른 팀과의 연동 지점을 파악할 때 활용
- 워크샵 중 "이런 이벤트는 어떻게 표현하나요?" 질문에 대한 답변 참고

### 1.2 포스트잇 색상 안내

```
┌─────────────────────────────────────────────────────────────┐
│                   포스트잇 색상 가이드                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  🟧 오렌지    도메인 이벤트 ("~되었다" 과거형)              │
│  🟦 파란색    커맨드 ("~하기" 명령형)                       │
│  🟨 노란색    애그리게이트 (데이터 모델)                    │
│  💜 보라색    정책/비즈니스 규칙 ("만약 ~하면")             │
│  🩷 분홍색    핫스팟 (문제/논의 필요)                       │
│  🟩 연두색    외부 시스템                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 전시팀 이벤트 특성 요약

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤

│  전시팀은 데이터 소비/조합자로서:                          │
│                                                             │
│  ❌ 이벤트가 아닌 것:                                      │
│  • API 호출/응답 처리                                      │

│  • 실시간 데이터 조합/변환                                 │
│  • 화면 렌더링 처리                                        │
│                                                             │
│  ✅ 이벤트인 것:                                           │
│  • 운영자/MD의 전시 구성 변경                              │
│  • 배치 처리 완료 결과                                     │

│  • 예약된 전시 시작/종료                                   │
│  • 서비스 영향 있는 장애 상황                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
