#### PDSS 이벤트 전파 흐름 예시

```
┌─────────────────────────────────────────────────────────────────┐
│                  PDSS 채널 등록 이벤트 흐름                      │
└─────────────────────────────────────────────────────────────────┘

[시스템 관리자] 🧑‍💼
   │
   ▼
🟦 채널 등록하기
   │
   ▼  🟨
🟧 채널 ─ 채널이 등록되었다
   │      - 채널ID: CH003
   │      - 채널명: GS MY SHOP
   │      - 채널유형: 모바일
   │
   ├──→ 💜 "채널 등록 시 기본 시간대 정책 자동 생성"
   │         │
   │         ▼
   │    🟧 시간대 정책이 등록되었다
   │
   ├──→ 편성: 새 채널 편성표 생성 가능
   ├──→ 온에어: 새 채널 읽기 모델 준비
   ├──→ PGM: 새 채널 정산 기준 설정
   └──→ 타사 모니터: 비교 기준 채널 추가
```

### 5.4 온에어 서비스

```
┌─────────────────────────────────────────────────────────────────┐
│                      온에어 서비스                               │
│                      약어: BOM │ 시간축: 방송 중                │
│                      ⭐ Read 위주 서비스                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  🟧 도메인 이벤트 (Write — 소수)                               │
│  ─────────────────────────────────────────────────────────────  │
│  • 방송이 시작되었다                                           │
│  • 방송 중 상품이 교체되었다                                   │
│  • 방송 중 긴급 공지가 발행되었다                              │
│  • 방송이 종료되었다                                           │
│  • 방송이 연장되었다                                           │
│                                                                 │
│  📖 읽기 모델 (Read — 주요 역할)                               │
│  ─────────────────────────────────────────────────────────────  │
│  • 편성정보 읽기모델 (← 편성 확정 이벤트)                      │
│  • 방송상품정보 읽기모델 (← 🟩 상품 서비스)                   │
│  • 실시간주문현황 읽기모델 (← 🟩 주문 서비스)                 │
│  • 실시간재고 읽기모델 (← 🟩 상품 서비스)                     │
│  • 실시간시청률 읽기모델 (← 시청률 데이터)                     │
│                                                                 │
│  🟦 커맨드 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • 방송 시작하기                                               │
│  • 방송 중 상품 교체하기                                       │
│  • 긴급 공지 발행하기                                          │
│  • 방송 종료하기                                               │
│  • 방송 연장하기                                               │
│                                                                 │
│  🟨 애그리게잇 예시                                            │
│  ─────────────────────────────────────────────────────────────  │
│  • 방송                                                        │
│  • 방송상품                                                    │
│                                                                 │
│  🟩 외부 시스템 연동                                           │
│  ─────────────────────────────────────────────────────────────  │
│  • 상품 서비스 (상품 정보, 재고, 가격)                         │
│  • 주문 서비스 (주문 현황, 결제)                               │
│  • 편성 서비스 (편성 확정 정보)                                │
│  • 전시 서비스 (노출/검색 연동)                                │
│                                                                 │
│  💜 정책 예시                                                  │
│  ─────────────────────────────────────────────────────────────  │
│  • "편성 확정 이벤트 수신 시 읽기 모델 자동 생성"              │
│  • "방송 종료 시 PGM 실적/정산 서비스에 데이터 전달"           │
│  • "방송 중 상품 교체 시 주문/상품 서비스에 알림"              │
│  • "편성 서비스 장애 시 기존 읽기 모델로 운영"                 │

│  🩷 핫스팟 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • 장애 격리 전략 (편성 장애 시 온에어 운영)                   │
│  • 읽기 모델 갱신 지연 허용 범위                               │
│  • 내부 시스템(상품/주문) 장애 시 대응                         │
│  • 실시간 데이터 동기화 방식 (이벤트 vs 폴링)                  │

└─────────────────────────────────────────────────────────────────┘
```

### 6.3 이벤트 소유권 원칙 (채널 vs 내부)

```
┌─────────────────────────────────────────────────────────────────┐
│              이벤트 소유권: 채널 시스템 vs 내부 시스템           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  원칙: "데이터를 변경하는 시스템이 이벤트를 발행한다"           │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  채널 시스템이 발행하는 이벤트 (채널 소유):                     │
│  ┌────────────────────────────────────────────────────────┐    │
│  │ 서비스    │ 이벤트                                     │    │
│  ├───────────┼────────────────────────────────────────────┤    │
│  │ PDSS      │ 채널 등록/변경/비활성화                    │    │
│  │ 편성      │ 편성표 확정/변경/긴급편성                  │    │
│  │ 기획      │ 기획서 승인/반려, 출연자 확정              │    │
│  │ 온에어    │ 방송 시작/종료, 상품 교체                  │    │
│  │ 타사모니터│ 타사정보 수집 완료                         │    │
│  │ 매체      │ 매체비용 정산                              │    │
│  │ PGM실적   │ 방송실적 집계, 정산 확정                   │    │
│  └───────────┴────────────────────────────────────────────┘    │
│                                                                 │
│  내부 시스템이 발행하는 이벤트 (내부 소유):                     │
│  ┌────────────────────────────────────────────────────────┐    │
│  │ 서비스    │ 이벤트                                     │    │
│  ├───────────┼────────────────────────────────────────────┤    │
│  │ 상품팀    │ 상품가격 변경, 재고 변경                   │    │
│  │ 주문팀    │ 주문 접수, 결제 완료                       │    │
│  │ 전시팀    │ 노출 변경, 검색 색인                       │    │
│  └───────────┴────────────────────────────────────────────┘    │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  워크샵에서의 표현 규칙:                                       │
│  • 채널 소유 이벤트 → 🟧 오렌지 포스트잇                      │
│  • 내부 시스템 이벤트 → 🟩 초록색 포스트잇 (외부 시스템)      │
│  • 소유권 불명확 → 🩷 핫스팟 (분홍색)                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
