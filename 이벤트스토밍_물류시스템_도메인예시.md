## 5. 물류 서비스별 이벤트 예시

### 5.1 배송관리 (FU — Fulfillment)

```
┌─────────────────────────────────────────────────────────────────┐
│                    배송관리 서비스 (Fulfillment)                 │
│                    약어: FU │ 프로세스축: 출하~배송완료         │
│                    ⭐ 물류 시스템의 핵심 허브                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  🟧 도메인 이벤트                                              │
│  ─────────────────────────────────────────────────────────────  │
│  • 배송정보가 전달되었다                                       │
│  • 배송구분이 판단되었다                                       │
│  • 출하지시가 생성되었다                                       │
│  • 출하지시가 취소되었다                                       │
│  • 배송지시정보가 전송되었다                                   │
│  • 배송정보 중복이 감지되었다                                  │
│  • 마감배치가 실행되었다                                       │
│  • 회수지시가 전달되었다                                       │
│  • 센터경유 출하지시가 생성되었다                              │
│  • 배송정보 재처리가 완료되었다                                │
│                                                                 │
│  🟦 커맨드 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • 배송정보 전달하기                                           │
│  • 배송구분 판단하기                                           │
│  • 출하지시 생성하기                                           │
│  • 출하지시 취소하기                                           │
│  • 마감배치 실행하기                                           │
│  • 배송정보 재처리하기                                         │
│  • 센터경유 출하지시 생성하기                                  │
│  • 회수지시 전달하기                                           │
│                                                                 │
│  🟨 애그리게잇 예시                                            │
│  ─────────────────────────────────────────────────────────────  │
│  • 출하지시                                                    │
│  • 배송정보                                                    │
│  • 배송구분                                                    │
│                                                                 │
│  💜 정책 예시                                                  │
│  ─────────────────────────────────────────────────────────────  │
│  • "센터배송여부에 따라 WMS 출고 또는 택배사 직송 분기"       │
│  • "마감시간 기준 배치 출하지시 생성"                          │
│  • "동일 키 값 동시요청 중복 방지 (DELIV_API_PROC_GB_CD)"     │
│  • "RetryException 시 재처리 대상 등록"                        │
│  • "센터경유 프로세스: 협력사 → 곤지암/안산 → 최종 배송"      │
│                                                                 │
│  🩷 핫스팟 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • 배송정보 동시요청 중복처리 방지 정합성                      │
│  • LIVE 배송 서비스 구축 시 FU 프로세스 변경 범위              │
│  • 도심거점 물류센터 추가 시 배송구분 확장                     │
│  • 센터경유 vs 직송 판단 기준 고도화                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### FU 이벤트 흐름 예시

```
┌─────────────────────────────────────────────────────────────────┐
│                  출하지시 생성 이벤트 흐름                       │
└─────────────────────────────────────────────────────────────────┘

🟩 기간계(OR): 주문이 완료되었다
   │
   ▼
🟦 배송정보 전달하기
   │
   ▼  🟨
🟧 배송정보 ─ 배송정보가 전달되었다
   │           - 주문번호: ORD2025031501
   │           - 배송지: 서울시 강남구
   │           - 상품정보: PRD001 x 2
   │
   ├──→ 💜 "동일 키 값 동시요청 중복 방지"
   │         │
   │         ├─ 중복 아님 → 진행
   │         └─ 중복 감지 → 🟧 배송정보 중복이 감지되었다
   │
   ▼
🟦 배송구분 판단하기
   │
   ▼  🟨
🟧 배송구분 ─ 배송구분이 판단되었다
   │           - 센터배송여부: Y
   │           - 배송유형: 일반택배
   │
   ├──→ 💜 "센터배송여부에 따라 분기"
   │         │
   │         ├─ [센터배송 Y] → WMS 출고지시
   │         └─ [직송 N] → COU 택배사 직접 지시
   │
   ▼
🟦 출하지시 생성하기
   │
   ▼  🟨
🟧 출하지시 ─ 출하지시가 생성되었다
   │           - 출하지시번호: SHP2025031501
   │           - 출고타입: 오늘도착
   │
   ├──→ WMS: 출고지시 접수
   └──→ COU: 택배사 지시정보 전송
```

### 5.2 창고관리 (WMS)

```
┌─────────────────────────────────────────────────────────────────┐
│                    창고관리 서비스 (WMS)                         │
│                    약어: WMS │ 프로세스축: 입고~출고            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  🟧 도메인 이벤트                                              │
│  ─────────────────────────────────────────────────────────────  │
│  • 출고지시가 접수되었다                                       │
│  • 출고처리가 완료되었다                                       │
│  • 입고가 확정되었다                                           │
│  • 반품입고가 확정되었다                                       │
│  • 양품화가 완료되었다                                         │
│  • 재고가 조정되었다                                           │
│  • 출고지시가 취소되었다                                       │
│  • 재고부족이 발생했다                                         │
│                                                                 │
│  🟦 커맨드 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • 출고지시 접수하기                                           │
│  • 출고처리 완료하기                                           │
│  • 입고 확정하기                                               │
│  • 반품입고 확정하기                                           │
│  • 양품화 처리하기                                             │
│  • 재고 조정하기                                               │
│                                                                 │
│  🟨 애그리게잇 예시                                            │
│  ─────────────────────────────────────────────────────────────  │
│  • 출고                                                        │
│  • 입고                                                        │
│  • 재고                                                        │
│  • 양품화                                                      │
│                                                                 │
│  🟩 외부 시스템 연동                                           │
│  ─────────────────────────────────────────────────────────────  │
│  • WMS 솔루션 (피킹/패킹/적재 — WMS 내부)                     │
│  • 기간계 상품(IMS) (상품 마스터 정보 수신)                    │
│                                                                 │
│  💜 정책 예시                                                  │
│  ─────────────────────────────────────────────────────────────  │
│  • "출고타입(오늘도착/새벽도착)에 따른 마감시간 차등 적용"     │
│  • "재고 부족 시 자동 재고조정 후 FU 알림"                     │
│  • "반품입고 확정 후 양품화 프로세스 자동 시작"                │
│  • "다회용 포장재 사용 시 포장재 반납 추적"                    │
│                                                                 │
│  🩷 핫스팟 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • WMS 상품정보 I/F 누락 사례 (ITM.TITEM 속성 미변경)         │
│  • WMS 리뉴얼 시 인터페이스 변경 범위                          │
│  • 안산 거점(로지스밸리) 구축 시 WMS 연동 변경                 │
│  • 다회용 포장재 운영 프로세스 정의                            │
│  • 도심지역 물류센터 구축 시 WMS 분리 여부                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### WMS 이벤트 흐름 예시

```
┌─────────────────────────────────────────────────────────────────┐
│                    WMS 출고 이벤트 흐름                          │
└─────────────────────────────────────────────────────────────────┘

FU: 🟧 출하지시가 생성되었다 (센터배송=Y)
   │
   ▼
🟦 출고지시 접수하기
   │
   ▼  🟨
🟧 출고 ─ 출고지시가 접수되었다
   │       - 출하지시번호: SHP2025031501
   │       - 출고타입: 오늘도착
   │       - 거점: 곤지암센터
   │
   │  (WMS 내부: 피킹 → 패킹 → 검수)
   │
   ▼
🟧 출고처리가 완료되었다
   │  - 운송장번호: 6000123456789
   │  - 택배사: CJ대한통운
   │
   ├──→ FU: 출고완료 정보 수신
   ├──→ COU: 택배 인계 정보 전달
   └──→ STL: 포장비/물동량 데이터 전달


[반품 입고]

RTN: 🟧 회수가 확정되었다
   │
   ▼
🟦 반품입고 확정하기
   │
   ▼  🟨
🟧 입고 ─ 반품입고가 확정되었다
   │       - 반품번호: RTN2025031501
   │       - 상태: 양품/불량
   │
   ├──→ 💜 "반품입고 확정 후 양품화 프로세스 자동 시작"
   │         │
   │         ▼
   │    🟧 양품화가 완료되었다
   │       - 양품/불량 판정 결과
   │
   └──→ STL: 반품 물동량 데이터
```

### 5.3 택배사연동 (COU — Courier)

```

│                    택배사연동 서비스 (Courier)                   │

│  • 택배사 연동이 실패했다                                      │
│  • 편의점 배송이 접수되었다                                    │
│                                                                 │
│  🟦 커맨드 예시                                                │
│  ─────────────────────────────────────────────────────────────  │
│  • 택배사 지시정보 전송하기                                    │

│  • 배송정보 수신하기                                           │
│  • 택배사 회수요청 전송하기                                    │
│  • 편의점 배송 접수하기                                        │

│  🟨 애그리게잇 예시                                            │
│  ─────────────────────────────────────────────────────────────  │
│  • 배송지시                                                    │
│  • 운송장                                                      │
│  • 택배사정보                                                  │
│                                                                 │
│  🟩 외부 시스템 연동                                           │
│  ─────────────────────────────────────────────────────────────  │
│  • CJ대한통운 (GLS — 직접 API)                                │
```

```
   │         ├─ 한진/롯데 → EAI 전송

   │
   ▼

   │
   ▼

   │
   ▼

   │  - 배송상태: 집화/간선/배달출발/배달완료
   │
   └──→ TRK: 배송상태 추적 업데이트
```

```
│  • 회수 확정하기                                               │

│  • 편의점회수 접수하기                                         │
```

```
│  │ 회수지시 배치        │ RTN → FU         │ Batch    │ RTN      │        │
│  └──────────────────────┴──────────────────┴──────────┴──────────┘        │
│                                                                             │
│  배송추적 (TRK) 관련                                                       │
│  ┌──────────────────────┬──────────────────┬──────────┬──────────┐        │
│  │ 배송상태 수신        │ 택배사 → TRK     │ Real     │ TRK      │        │
│  │ 배송완료 전달        │ TRK → OR         │ Real     │ TRK      │        │
│  │ 출고SMS 발송         │ TRK → SMS        │ Real     │ TRK      │        │
│  │ 착지변경 전송        │ TRK → 택배사     │ Real     │ TRK      │        │
│  └──────────────────────┴──────────────────┴──────────┴──────────┘        │
│                                                                             │
│  물류정산 (STL) 관련                                                       │
│  ┌──────────────────────┬──────────────────┬──────────┬──────────┐        │
│  │ 정산 데이터 수집     │ FU/WMS/COU → STL │ Batch    │ STL      │        │
│  │ 물류전표 생성        │ STL → SAP        │ Batch    │ STL      │        │
│  │ SAP 전표 posting     │ STL → SAP        │ Batch    │ STL      │        │
│  └──────────────────────┴──────────────────┴──────────┴──────────┘        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.3 이벤트 소유권 원칙 (물류 vs 기간계 vs 외부시스템)

```
┌─────────────────────────────────────────────────────────────────┐
│         이벤트 소유권: 물류 vs 기간계 vs 외부시스템             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  원칙: "데이터를 변경하는 시스템이 이벤트를 발행한다"          │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  물류 시스템이 발행하는 이벤트 (물류 소유):                    │
│  ┌────────────────────────────────────────────────────────┐   │
│  │ 서비스    │ 이벤트                                     │   │
│  ├───────────┼────────────────────────────────────────────┤   │
│  │ FU        │ 배송정보 전달, 출하지시 생성/취소          │   │
│  │ WMS       │ 출고처리 완료, 입고 확정, 재고 조정        │   │
│  │ COU       │ 택배사 지시 전송, 배송정보 수신            │   │
│  │ RTN       │ 회수지시 생성, 회수 확정, 반품비 산정      │   │
│  │ TRK       │ 배송상태 변경, 배송 완료, 착지변경         │   │
│  │ STL       │ 물류정산 집계, 물류전표 생성               │   │
│  └───────────┴────────────────────────────────────────────┘   │
│                                                                 │
│  기간계가 발행하는 이벤트 (기간계 소유):                       │
│  ┌────────────────────────────────────────────────────────┐   │
│  │ 서비스    │ 이벤트                                     │   │
│  ├───────────┼────────────────────────────────────────────┤   │
│  │ 주문(OR)  │ 주문 완료, 반품 접수, 환불 처리            │   │
│  │ 상품(IMS) │ 상품정보 변경, 재고 변경                   │   │
│  └───────────┴────────────────────────────────────────────┘   │
│                                                                 │
│  외부 시스템 이벤트 (🟩):                                      │
│  ┌────────────────────────────────────────────────────────┐   │
│  │ 시스템    │ 이벤트                                     │   │
│  ├───────────┼────────────────────────────────────────────┤   │
│  │ WMS솔루션│ 피킹/패킹 완료 (WMS 내부)                  │   │
│  │ 택배사   │ 집화/간선/배달 (택배사 내부)                │   │
│  │ SAP      │ SAP 전표 posting 확정                      │   │
│  │ 편의점   │ CVSnet/BGFpost 접수/회수 완료              │   │
│  └───────────┴────────────────────────────────────────────┘   │
│                                                                 │
│  워크샵에서의 표현 규칙:                                       │
│  • 물류 소유 이벤트 → 🟧 오렌지 포스트잇                      │
│  • 기간계 이벤트 → 🟩 초록색 포스트잇 (외부 시스템)          │
│  • 외부 시스템 이벤트 → 🟩 초록색 포스트잇                    │
│  • 소유권 불명확 → 🩷 핫스팟 (분홍색)                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
