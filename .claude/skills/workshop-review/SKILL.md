# 워크샵 준비 문서 검증 스킬

## 입력

`$ARGUMENTS`에서 문서 파일 경로를 파싱한다.
- 예: `/workshop-review ./이벤트스토밍_검색서비스팀_2차워크샵준비.md`

## 검증 체크리스트

### 1. 파일 존재 검증

- [ ] 문서 파일(.md) 존재 확인
- [ ] 이미지 디렉토리(`images/{팀명}_{N차}/`) 존재 확인
- [ ] PNG 파일 10개 존재 확인:
  - `diagram_01_현황요약.png`
  - `diagram_02_흐름영역맵.png`
  - `diagram_03_타임라인.png`
  - `diagram_04_애그리게이트_비포애프터.png`
  - `diagram_05_영역별매핑.png`
  - `diagram_06_정책후보맵.png`
  - `diagram_07_정책이벤트연결.png` (또는 `핫스팟정책전환.png`)
  - `diagram_08_읽기모델후보.png`
  - `diagram_09_읽기모델이벤트연결.png`
  - `diagram_10_전체통합흐름.png`

### 2. 이미지 참조 정합성

문서 내 `![...](.../diagram_XX_...)` 패턴을 모두 추출하여:
- [ ] 참조된 이미지 파일이 실제로 존재하는지 확인
- [ ] 이미지 디렉토리 내 파일이 문서에서 참조되는지 확인 (고아 파일 감지)
- [ ] 상대 경로가 올바른지 확인 (`./images/{팀명}_{N차}/` 형식)

### 3. details 태그 검증

- [ ] `<details>` 블록 10개 존재 확인
- [ ] 각 `<details>` 안에 `<summary>📊 원본 Mermaid 코드 보기</summary>` 존재
- [ ] 각 `<details>` 안에 ` ```mermaid` 코드 블록 존재
- [ ] `</details>` 닫힘 태그 10개 존재
- [ ] 각 이미지 바로 아래에 대응하는 `<details>` 블록이 위치하는지 확인

### 4. 필수 섹션 검증

다음 10개 섹션이 모두 존재하는지 확인:

| # | 섹션 | 필수 키워드 |
|---|------|-----------|
| 1 | `## 1. 개요` | 목적, 목표, 참조 문서 |
| 2 | `## 2. *차 결과 정리 및 재검토` | 현황 요약, 흐름 영역 |
| 3 | `## 3. *차 워크샵 타임라인` | gantt, Phase |
| 4 | `## 4. Phase 1: 이벤트 정제` | 퍼실리테이터 스크립트, 재검토 대상 |
| 5 | `## 5. Phase 2: 애그리게이트 식별` | 눈높이 설명, Before/After |
| 6 | `## 6. Phase 3: 정책 도출` | 정책 후보, 이벤트 연결 |
| 7 | `## 7. Phase 4: 읽기 모델 도출` | 읽기 모델 후보, 이벤트 연결 |
| 8 | `## 8. 전체 통합 및 정리` | 통합 흐름, 성과 체크리스트 |
| 9 | `## 9. 퍼실리테이터 비상 대응 카드` | 상황, 대응 |
| 10 | `## 10. 결과물 템플릿` | 일시, 참석자 |

### 5. 포맷팅 규칙 검증

- [ ] **퍼실리테이터 스크립트**: `>` 인용 블록으로 작성되었는지 (최소 4개 Phase 스크립트)
- [ ] **체크리스트**: `- [ ]` 또는 `- [x]` 형식 사용 (최소 1개 섹션)
- [ ] **테이블**: 파이프(`|`) 구분 테이블 존재 (최소 5개)
- [ ] **ASCII 박스**: `┌─┐│└─┘` 패턴 사용 (개요 섹션)
- [ ] **이모지 상태 아이콘**: `✅`, `⬜` 사용
- [ ] **강조**: `**텍스트**` 형식 사용

### 6. Mermaid 다이어그램 검증

각 mermaid 코드 블록에 대해:
- [ ] 다이어그램 타입 확인 (pie, flowchart, gantt)
- [ ] classDef 정의가 표준 색상을 사용하는지 확인
- [ ] 노드에 class 지정이 올바른지 (`:::event`, `:::command` 등)
- [ ] 다이어그램 10개 타입 순서:
  1. `pie` — 현황 요약
  2. `flowchart LR` — 흐름 영역 맵
  3. `gantt` — 타임라인
  4. `flowchart TB` — 애그리게이트 Before/After
  5. `flowchart TB` + subgraph — 영역별 매핑
  6. `flowchart LR` — 정책 후보 맵
  7. `flowchart TB` 또는 `flowchart LR` — 핫스팟/정책 전환 또는 정책-이벤트 연결
  8. `flowchart TB` — 읽기모델 후보
  9. `flowchart LR` — 읽기모델-이벤트 연결
  10. `flowchart LR` + subgraph — 전체 통합

### 7. 내용 정합성 검증 (선택)

draw.io 분석 결과가 제공된 경우:
- [ ] 문서 내 이벤트 수와 분석 결과 이벤트 수 일치
- [ ] 문서 내 애그리게이트 수와 분석 결과 일치
- [ ] 정제 전/후 수치가 논리적으로 타당한지 확인

## 결과 보고서 형식

```markdown
# 문서 검증 결과: {파일명}

## 요약
- ✅ 통과: N개
- ❌ 실패: N개
- ⚠️ 경고: N개

## 상세 결과

### 1. 파일 존재 — ✅/❌
...

### 2. 이미지 참조 — ✅/❌
...

(각 항목별 상세 결과)

## 권장 수정 사항
1. ...
2. ...
```

## 실행 방법

1. Read 도구로 문서 파일을 읽는다.
2. Grep/Glob으로 이미지 파일 존재를 확인한다.
3. 각 체크리스트 항목을 순서대로 검증한다.
4. 결과 보고서를 출력한다.
