# 워크샵 준비 문서 섹션 구조 템플릿

## 문서 제목

```
# {팀명 (띄어쓰기)} 이벤트 스토밍 {N차} 워크샵 준비 가이드
```

예시:
- `# 검색 서비스팀 이벤트 스토밍 2차 워크샵 준비 가이드`
- `# 주문 서비스팀 이벤트 스토밍 3차 워크샵 준비 가이드`

---

## 섹션 1: 개요 (`## 1. 개요`)

### 하위 섹션
- `### 1.1 이 문서의 목적` — 문서 목적 1문장 + ASCII 박스(목표 체크리스트)
- `### 1.2 {N-1}차 요약 & {N차} 목표` — 이전 차수 완료/현재 목표 테이블
- `### 1.3 참조 문서` — 참조 문서 링크 테이블

### 포함 다이어그램
없음

### 차수별 분기
- **2차**: `### 1.2 1차 요약 & 2차 목표`
- **3차**: `### 1.2 1~2차 요약 & 3차 목표` — 1~2차 완료 사항을 별도 텍스트로 먼저 기술

### 필수 요소
- ASCII 박스: `┌─┐│└─┘` 패턴으로 목표 리스트
- 요약 테이블: 항목 | 이전 완료 | 현재 목표 (✅/⬜ 아이콘)
- 참조 문서 테이블: 문서명(링크) | 활용 시점/용도

---

## 섹션 2: 이전 결과 정리 및 재검토 (`## 2. {N-1}차 결과 정리 및 재검토`)

### 하위 섹션
- `### 2.1 현재 요소 현황 요약` — pie 다이어그램 + 현황 분석
- `### 2.2 {M}개 흐름 영역 전체 맵` — 흐름 영역 다이어그램 + 영역별 상세

### 포함 다이어그램
- **다이어그램 1**: 현황 요약 (pie)
- **다이어그램 2**: 흐름 영역 맵 (flowchart LR)

### 차수별 분기
- **2차**: `## 2. 1차 결과 정리 및 재검토`
- **3차**: `## 2. 1~2차 결과 정리 및 재검토`

### 필수 요소
- 현황 분석 불렛 포인트 (3~5개)
- 흐름 영역 맵 아래 각 영역별 간략 설명

---

## 섹션 3: 워크샵 타임라인 (`## 3. {N차} 워크샵 타임라인`)

### 하위 섹션
없음 (gantt 다이어그램 + 상세 테이블)

### 포함 다이어그램
- **다이어그램 3**: 타임라인 (gantt)

### 필수 요소
- gantt 다이어그램
- 상세 시간표 테이블: 시간 | 단계 | 활동 | 산출물
- Phase별 시간 배분 근거 설명 (선택)

---

## 섹션 4: Phase 1 — 이벤트 정제 (`## 4. Phase 1: 이벤트 정제 ({M}분)`)

### 하위 섹션
- `### 퍼실리테이터 스크립트` — `>` 인용 블록
- `### 재검토 대상 처리 가이드` — 통합/제외/재분류 대상
- `### 정제 후 예상 이벤트 목록 (~{N}개)` — 전체 이벤트 테이블

### 포함 다이어그램
없음 (섹션 2의 다이어그램을 참조)

### 차수별 분기
- **2차**: 시간 20분, 정제 규모 보통
- **3차**: 시간 30분, "대폭 정제" 강조, 오분류 교정 포함

### 필수 요소
- 퍼실리테이터 스크립트 (대화체, 구체적 수치 포함)
- 통합 대상: 번호 리스트 + `→ **"통합 이벤트명"**`
- 제외 대상: 번호 리스트 + `(→ 사유)`
- 재분류 대상: 번호 리스트 + `→ 📖/💜/🟦 변환 대상`
- 정제 후 이벤트 테이블: # | 영역 | 이벤트

---

## 섹션 5: Phase 2 — 애그리게이트 식별 (`## 5. Phase 2: 애그리게이트 식별 ({M}분)`)

### 하위 섹션
- `### 5.1 {팀명} 눈높이 설명` — 팀 특화 비유로 애그리게이트 설명
- `### 5.2 기존 라벨 재검토` 또는 `### 5.2 Before/After` — 오분류 분석
- `### 5.3 {N}개 후보 상세` 또는 유사 — 애그리게이트 후보 상세 설명
- 퍼실리테이터 스크립트 (인용 블록)

### 포함 다이어그램
- **다이어그램 4**: 애그리게이트 Before/After (flowchart TB)
- **다이어그램 5**: 영역별 매핑 (flowchart TB + subgraph)

### 필수 요소
- 눈높이 설명: 팀 도메인에 친숙한 비유 (검색팀→ES 인덱스, 주문팀→주문서 등)
- Before/After 비교
- 애그리게이트별 상세: 이름, 포함 데이터, 관련 이벤트, 비즈니스 규칙

---

## 섹션 6: Phase 3 — 정책 도출 (`## 6. Phase 3: 정책 도출 ({M}분)`)

### 하위 섹션
- `### 6.1 {팀명} 눈높이 설명` — 정책 개념 설명
- `### 6.2 정책 후보 목록` — 정책 후보 상세
- 퍼실리테이터 스크립트

### 포함 다이어그램
- **다이어그램 6**: 정책 후보 맵 (flowchart LR)
- **다이어그램 7**: 핫스팟/정책 전환 또는 정책-이벤트 연결

### 차수별 분기
- **2차**: 다이어그램 7은 "정책-이벤트 연결"
- **3차**: 다이어그램 7은 "핫스팟→정책 전환" (기존 핫스팟을 정책으로 전환)

### 필수 요소
- 눈높이 설명 (인용 블록)
- 정책별 상세: 정책명, 트리거 이벤트, 조건, 결과 액션
- 정책 테이블: # | 정책 | 트리거 | When 조건 | Then 결과

---

## 섹션 7: Phase 4 — 읽기 모델 도출 (`## 7. Phase 4: 읽기 모델 도출 ({M}분)`)

### 하위 섹션
- `### 7.1 {팀명} 눈높이 설명` — 읽기 모델 개념 설명
- `### 7.2 읽기 모델 후보 목록` — 읽기 모델 상세
- 퍼실리테이터 스크립트

### 포함 다이어그램
- **다이어그램 8**: 읽기 모델 후보 (flowchart TB)
- **다이어그램 9**: 읽기모델-이벤트 연결 (flowchart LR)

### 필수 요소
- 눈높이 설명 (인용 블록)
- 읽기모델별 상세: 모델명, 소비자, 포함 데이터, 갱신 트리거 이벤트
- 용도별 분류: 고객/사용자 화면 vs 운영자/관리자 화면

---

## 섹션 8: 전체 통합 및 정리 (`## 8. 전체 통합 및 정리 ({M}분)`)

### 하위 섹션
- 전체 통합 흐름 다이어그램
- 성과 체크리스트 (목표 달성 확인)
- 다음 차수 예고

### 포함 다이어그램
- **다이어그램 10**: 전체 통합 흐름 (flowchart LR + subgraph)

### 필수 요소
- 통합 다이어그램 (모든 요소 포함)
- 성과 체크리스트: `- [ ]` 형식
- 워크샵 성과 요약 테이블: 항목 | 목표 | 달성 (✅/⬜)
- 다음 단계 안내

---

## 섹션 9: 퍼실리테이터 비상 대응 카드 (`## 9. 퍼실리테이터 비상 대응 카드`)

### 필수 요소
- 3~5개 비상 상황 + 대응 방법
- 각 상황: `**상황:**` + `**대응:**` 형식
- 팀 특화 상황 포함 (예: 검색팀 - 기술 논쟁, 주문팀 - 레거시 논쟁)

---

## 섹션 10: 결과물 템플릿 (`## 10. 결과물 템플릿`)

### 필수 요소
- 워크샵 결과물 기록용 빈 템플릿
- 포함 항목: 일시, 참석자, 이벤트 정제 결과, 애그리게이트, 정책, 읽기 모델, 핫스팟/미결 사항, 다음 단계
- 각 항목에 빈 테이블 또는 기입란
