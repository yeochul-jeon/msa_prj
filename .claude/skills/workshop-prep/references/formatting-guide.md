# 워크샵 준비 문서 포맷팅 규칙

## 1. 이미지 삽입 패턴

다이어그램은 반드시 다음 패턴으로 삽입한다:

```markdown
![다이어그램 제목](./images/{팀명}_{N차}/diagram_XX_이름.png)

<details>
<summary>📊 원본 Mermaid 코드 보기</summary>

` ` `mermaid
(mermaid 코드)
` ` `

</details>
```

**규칙:**
- 이미지 링크와 `<details>` 사이에 빈 줄 1개
- `<details>` 시작 태그와 `<summary>` 사이에 줄바꿈 없음
- `</summary>` 뒤에 빈 줄 1개
- mermaid 코드 블록 뒤에 빈 줄 1개
- `</details>` 뒤에 빈 줄 1개

## 2. 퍼실리테이터 스크립트

`>` 인용 블록으로 작성한다. 대화체로, 워크샵 현장에서 바로 읽을 수 있는 형태.

```markdown
### 퍼실리테이터 스크립트

> "1차에서 40개의 이벤트를 도출했습니다. 오늘 첫 단계로, 중복되거나 비즈니스 이벤트가 아닌 것들을 정리하겠습니다.
> 화면에 재검토 대상 8건을 띄워놓았으니, 각각에 대해 '통합할지, 제외할지, 재분류할지' 빠르게 결정하겠습니다.
> 한 건당 2분을 넘기지 않겠습니다."
```

**규칙:**
- 쌍따옴표로 전체를 감싼다
- 각 줄은 `> `로 시작
- 구체적 수치를 포함 (예: "40개", "8건", "2분")
- `**강조**`를 적절히 사용
- 3~5줄 분량

### 개념 설명용 스크립트

```markdown
> **애그리게이트 = "함께 변하는 데이터 묶음"**
>
> 검색팀에 친숙한 비유로 설명하면:
>
> - **Elasticsearch 인덱스 하나** = 하나의 애그리게이트와 비슷합니다
> - 인덱스에 들어있는 문서들은 함께 색인되고, 함께 조회되고, 함께 갱신됩니다
> - "이 데이터가 변할 때 같이 변해야 하는 데이터가 뭐가 있지?" → 그게 애그리게이트입니다
```

**규칙:**
- 첫 줄: `**개념 = "한 줄 정의"**`
- 빈 인용줄(`>`)로 단락 구분
- 팀 도메인 비유 필수 포함
- 질문-답변 형식 활용 (`"~가 뭐지?" → 그게 ~입니다`)

## 3. ASCII 박스

개요 섹션의 목표 체크리스트에 사용한다:

```
┌─────────────────────────────────────────────────────────────┐
│              {N차} 워크샵에서 달성할 것                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ✅ 목표 1                                                 │
│  ✅ 목표 2                                                 │
│  ✅ 목표 3                                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**규칙:**
- 코드 블록(` ``` `)으로 감싼다
- 상자 문자: `┌ ─ ┐ │ ├ ┤ └ ┘`
- 너비: ~60자 기준
- 각 항목 앞에 `✅` 이모지
- 상하 여백 1줄씩

## 4. 테이블

파이프(`|`) 구분 마크다운 테이블을 사용한다:

### 요약/비교 테이블
```markdown
| 항목 | 이전 완료 | 현재 목표 |
|------|---------|---------|
| 이벤트 도출 | ✅ 40개 | 정제하여 ~30개로 통합 |
| 애그리게이트 | ⬜ 초안 7개 | **10개 후보 확정** |
```

### 이벤트 목록 테이블
```markdown
| # | 영역 | 이벤트 |
|---|------|--------|
| 1 | ① 검색 진입 | 추천 상품 목록이 노출됨 |
```

### 참조 문서 테이블
```markdown
| 참조 문서 | 활용 시점 |
|----------|----------|
| [문서명](./파일명.md) | 설명 |
```

### 정책 테이블
```markdown
| # | 정책 | 트리거 | When 조건 | Then 결과 |
|---|------|--------|----------|----------|
| 1 | 정책명 | 이벤트 | 조건 | 결과 |
```

**규칙:**
- 헤더와 구분선 포함
- 상태 아이콘: `✅` (완료), `⬜` (미완료)
- 강조가 필요한 목표 항목: `**굵게**`
- 영역 번호: `①②③④⑤⑥⑦⑧⑨⑩` 원문자 사용

## 5. 체크리스트

성과 확인, 워크샵 준비물 등에 사용:

```markdown
### 성과 체크리스트

- [ ] 이벤트 정제: 40개 → ~30개
- [ ] 애그리게이트: 10개 후보 확정
- [ ] 정책: 6개 후보 도출
- [ ] 읽기 모델: 6개 후보 도출
- [ ] 바운디드 컨텍스트 후보 프리뷰
```

## 6. 구분선

섹션 간 구분에 `---` 사용:

```markdown
---

## 3. 타임라인
```

**규칙:**
- `---` 전후로 빈 줄 1개씩
- 주요 섹션(##) 전에 사용
- 하위 섹션(###) 사이에는 사용하지 않음

## 7. 숫자 표기

- 이벤트 수 등 정량 데이터: `40개`, `~30개`
- 범위/변환: `40개 → ~30개`, `7개 → 10개 후보`
- 시간: `20분`, `09:00`, `(30분)`
- 차수: `1차`, `2차`, `N차`

## 8. 이모지 사용 규칙

문서 내에서 사용하는 이모지는 이벤트 스토밍 요소 표기용으로 제한:

| 이모지 | 용도 |
|--------|------|
| 🟧 | 도메인 이벤트 |
| 🟦 | 커맨드 |
| 🟨 | 애그리게이트 |
| 💜 | 정책 |
| 🩷 | 핫스팟 |
| 🟩 | 외부 시스템 |
| 📖 | 읽기 모델 |
| ✅ | 완료/달성 |
| ⬜ | 미완료/예정 |
| 📊 | Mermaid 코드 details 태그 |
| 👤 | 고객/사용자 |
| 🔧 | 운영자/관리자 |
