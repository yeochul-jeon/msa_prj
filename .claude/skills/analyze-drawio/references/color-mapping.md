## 색상 매칭 로직

색상 비교 시 정확한 코드 일치가 아니라 **유사 색상 범위**로 매칭한다:

1. HEX 코드를 RGB로 변환
2. 각 색상 범위의 대표 색상과의 거리를 계산
3. 가장 가까운 색상 범위로 매핑
4. 거리가 임계값(50)을 초과하면 "미분류"

간편 매칭 (fillColor 값 직접 비교):
- `#FF` 시작 + G채널 `80~A0` 범위 → 오렌지 (이벤트)
- `#FF` 시작 + G채널 `60~70` + B채널 `B0~` → 핑크 (핫스팟)
- `#1~4` 시작 + B채널 높음 → 파랑 (커맨드)
- `#FF` 시작 + G채널 `D0~EB` → 노랑 (애그리게이트)
- `#9C`, `#80`, `#6A`, `#7B` 시작 → 보라 (정책)
- `#2E`, `#4C`, `#38` 시작 + G채널 높음 → 초록 (외부시스템)
- `#E3F2`, `#87CE`, `#BBDE`, `#90CA` 시작 → 하늘 (읽기모델)

## 오분류 감지 규칙

### 이름 패턴 기반 감지

| 현재 색상(분류) | 이름 패턴 | 의심 사유 | 제안 분류 |
|----------------|----------|----------|----------|
| 🟨 노랑 (애그리게이트) | `~됨`, `~실패`, `~발송`, `~완료됨` | 이벤트 이름 패턴 | 🟧 이벤트 |
| 🟧 오렌지 (이벤트) | `~조회`, `~목록`, `~현황` | 읽기모델/커맨드 패턴 | 📖 읽기모델 또는 🟦 커맨드 |
| 🟨 노랑 (애그리게이트) | `~검증`, `~확인`, `~판단` | 정책 패턴 | 💜 정책 |
| 🟧 오렌지 (이벤트) | `~하다`, `~요청하다`, `~실행하다` | 커맨드 패턴 (현재형) | 🟦 커맨드 |
| 🟦 파랑 (커맨드) | `~됨`, `~되었다`, `~발생함` | 이벤트 패턴 (과거형) | 🟧 이벤트 |

### 구조 기반 감지

- 같은 색상 포스트잇이 10개 이상 → 세분화 필요 경고
- 이벤트가 없는데 커맨드만 있는 영역 → 이벤트 누락 경고
- 애그리게이트가 이벤트보다 많은 영역 → 과도한 분리 경고
