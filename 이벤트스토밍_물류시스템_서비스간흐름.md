## 1. 전체 배송 라이프사이클 흐름

### 1.1 주문~배송~반품 전체 흐름도

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    배송 라이프사이클 전체 흐름                               │
└─────────────────────────────────────────────────────────────────────────────┘

  기간계(주문)          배송관리(FU)           택배사연동(COU)        배송추적(TRK)
 ─────────────────────────────────────────────────────────────────────────────

  ┌──────────┐        ┌──────────────┐        ┌──────────────┐     ┌──────────┐
  │ 🟩 주문  │ ─────→ │  배송관리    │ ─────→ │ 택배사연동   │ ──→ │ 배송추적 │
  │ 서비스   │        │   (FU)       │        │   (COU)      │     │  (TRK)   │
  │ (OR)     │        │ 출하지시     │        │ 택배사별     │     │ 배송상태 │
  └──────────┘        │ 배송구분     │        │ 지시/수신    │     │ 착지변경 │
                      └──────┬───────┘        └──────────────┘     └─────┬────┘
                             │                                           │
                    센터배송여부 분기                                     │
                    ┌────────┴────────┐                                  │
                    │ Y              │ N                                 │
                    ▼                ▼                                   ▼
             ┌──────────┐    (택배사 직송)                        ┌──────────┐
             │ 창고관리 │                                         │ 🟩 주문  │
             │  (WMS)   │                                         │ 서비스   │
             │ 입고/출고│                                         │ (상태    │
             │ 재고관리 │                                         │  업데이트)│
             └──────────┘                                         └──────────┘


  반품/회수(RTN)         배송관리(FU)           택배사연동(COU)
 ─────────────────────────────────────────────────────────────────────────────

  ┌──────────┐        ┌──────────────┐        ┌──────────────┐
  │ 🟩 주문  │ ─────→ │  반품/회수   │ ─────→ │  배송관리    │ ──→ 택배사연동
  │ 서비스   │        │   (RTN)      │        │   (FU)       │     (COU)
  │ 반품접수 │        │ 회수지시     │        │ 회수지시     │
  └──────────┘        │ 회수확정     │        │ 전달         │
                      └──────┬───────┘        └──────────────┘
                             │
                             ▼
                      ┌──────────────┐        ┌──────────────┐
                      │  창고관리    │ ─────→ │  물류정산    │
                      │   (WMS)      │        │   (STL)      │
                      │  입고확정    │        │ 배송비/포장비│
                      │  양품화      │        │ SAP 전표     │
                      └──────────────┘        └──────────────┘


 ─────────────────────────────────────────────────────────────────────────────
 범례:
 ──→  이벤트 흐름 (주방향)
 🟩   외부 시스템 (기간계 OR, 상품 IMS)
```

### 1.2 프로세스축별 주요 이벤트 타임라인

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                  배송 프로세스 이벤트 타임라인                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  주문 완료 (T+0)                                                           │
│  ─────────────────────────────────────────────────────────────────────────  │
│  FU   │ 🟧 배송정보가 전달되었다 / 출하지시가 생성되었다                   │
│  FU   │ 🟧 배송구분이 판단되었다 (센터/직송/편의점)                        │
│       │                                                                     │
│  출하~출고 (T+0 ~ T+1)                                                     │
│  ─────────────────────────────────────────────────────────────────────────  │
│  WMS  │ 🟧 출고지시가 접수되었다 / 출고처리가 완료되었다                   │
│  COU  │ 🟧 택배사 지시정보가 전송되었다 / 운송장이 발급되었다             │
│  TRK  │ 🟧 출고SMS가 발송되었다                                            │
│       │                                                                     │
│  배송 중 (T+1 ~ T+3)                                                       │
│  ─────────────────────────────────────────────────────────────────────────  │
│  COU  │ 🟧 배송정보가 수신되었다 (택배사 → 시스템)                         │
│  TRK  │ 🟧 배송상태가 변경되었다 (집화/간선/배달출발/배달완료)             │
│  TRK  │ 🟧 착지변경이 요청되었다                                           │
│       │                                                                     │
│  배송 완료 (T+3 ~ T+5)                                                     │
│  ─────────────────────────────────────────────────────────────────────────  │
│  TRK  │ 🟧 배송이 완료되었다                                               │
│  STL  │ 🟧 배송비가 정산되었다                                             │
│       │                                                                     │
│  반품/회수 (T+5 ~ T+15)                                                    │
│  ─────────────────────────────────────────────────────────────────────────  │
│  RTN  │ 🟧 반품이 접수되었다 / 회수지시가 생성되었다                       │
│  RTN  │ 🟧 회수가 확정되었다 / 환불이 처리되었다                           │
│  WMS  │ 🟧 반품입고가 확정되었다 / 양품화가 완료되었다                     │
│       │                                                                     │
│  정산 (월말)                                                                │
│  ─────────────────────────────────────────────────────────────────────────  │
│  STL  │ 🟧 월별 물류정산이 집계되었다 / 물류전표가 생성되었다             │
│  STL  │ 🟧 SAP 전표가 posting되었다                                        │
│       │                                                                     │
│  범례: 🟧 도메인 이벤트                                                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.3 이벤트 발행/구독 매트릭스 (6×6)

> 6개 서비스 간 이벤트 발행(Publish)과 구독(Subscribe) 관계를 매트릭스로 표현합니다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    이벤트 발행/구독 매트릭스 (6×6)                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  발행 ＼ 구독    FU     WMS    COU    RTN    TRK    STL                    │
│  ──────────────────────────────────────────────────────────────────         │
│  FU(배송관리)  │  -     ●      ●      -      ●      ●                     │
│  WMS(창고관리) │  ●     -      -      ●      -      ●                     │
│  COU(택배사)   │  -     -      -      -      ●      ●                     │
│  RTN(반품/회수)│  ●     ●      ●      -      -      ●                     │
│  TRK(배송추적) │  -     -      -      -      -      -                     │
│  STL(물류정산) │  -     -      -      -      -      -                     │
│                                                                             │
│  ● = 주요 이벤트 구독 (필수 연동)                                           │
│                                                                             │
│  핵심 흐름:                                                                 │
│  🟩 기간계(OR) → FU    : 주문완료 → 배송정보전달 (가장 중요한 시작점)      │
│  FU → WMS/COU         : 출하지시 → 센터출고 또는 택배사직송                │
│  COU → TRK            : 택배사 배송정보 → 배송상태 추적                    │
│  🟩 기간계(OR) → RTN   : 반품접수 → 회수지시 생성                          │
│  RTN → FU → COU       : 회수지시 → 택배사 회수 요청                        │
│  FU/WMS/COU → STL     : 배송/출고/택배 데이터 → 정산                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 2. 서비스 간 이벤트 흐름 상세

### 2.1 기간계(OR) → FU (주문→배송정보전달)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    기간계(OR) → 배송관리(FU) 흐름                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ⭐ 물류 시스템의 시작점 — 주문 완료 후 배송 프로세스 진입                  │
│                                                                             │
│  ┌──────────────┐                        ┌──────────────┐                  │
│  │ 🟩 기간계    │                        │  배송관리    │                  │
│  │   (OR)       │                        │   (FU)       │                  │
│  │ 주문 서비스  │                        │ 출하지시     │                  │
│  └──────┬───────┘                        └──────┬───────┘                  │
│         │                                        │                          │
│         │ 🟩 주문이 완료되었다                    │                          │
│         │   - 주문번호, 배송지, 상품정보           │                          │
│         │────────────────────────────────────→   │                          │
│         │                                        │                          │
│         │                                        │ 🟧 배송정보가            │
│         │                                        │    전달되었다            │
│         │                                        │                          │
│         │                                        │ 💜 배송구분 자동 판단    │
│         │                                        │    (센터/직송/편의점)    │
│         │                                        │                          │
│         │                                        │ 🟧 출하지시가            │
│         │                                        │    생성되었다            │
│                                                                             │
│  연동 방식:                                                                 │
│  • EAI 기반 (Real-time, Async)                                             │
│  • condition-api / interlink-api 경유                                      │
│                                                                             │
│  💜 정책:                                                                   │
│  • "동일 키 값(회수지, 출고지, 배송비 정책) 동시요청 중복 방지"            │
│  • "마감시간 기준 배치 출하지시 생성"                                       │
│                                                                             │
│  🩷 핫스팟:                                                                 │
│  • 배송정보 동시요청 중복처리 (DELIV_API_PROC_GB_CD 관리)                  │
│  • RetryException 기반 재처리 시 정합성                                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 FU → WMS (출하지시→출고처리)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    배송관리(FU) → 창고관리(WMS) 흐름                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  센터배송(곤지암, 안산 등) 경우에만 WMS 연동                               │
│                                                                             │
│  ┌──────────────┐                        ┌──────────────┐                  │
│  │  배송관리    │                        │  창고관리    │                  │
│  │   (FU)       │                        │   (WMS)      │                  │
│  └──────┬───────┘                        └──────┬───────┘                  │
│         │                                        │                          │
│         │ 🟧 출하지시가 생성되었다                │                          │
│         │   - 센터배송 = Y                        │                          │
│         │   - 출고타입(오늘도착/새벽도착)          │                          │
│         │────────────────────────────────────→   │                          │
│         │                                        │ 🟧 출고지시가            │
│         │                                        │    접수되었다            │
│         │                                        │                          │
│         │                                        │ (피킹/패킹/출고)        │
│         │                                        │                          │
│         │    🟧 출고처리가 완료되었다              │                          │
│         │   ←────────────────────────────────────│                          │
│         │                                        │                          │
│                                                                             │
│  연동 포인트:                                                               │
│  • FU → WMS: "이 주문의 상품을 출고하라"                                   │
│  • WMS → FU: "출고 완료, 운송장 정보 전달"                                 │
│                                                                             │
│  💜 정책:                                                                   │
│  • "출고타입(오늘도착/새벽도착)에 따른 마감시간 차등 적용"                  │
│  • "재고 부족 시 자동 재고조정 후 알림"                                    │
│                                                                             │
│  🩷 핫스팟:                                                                 │
│  • WMS 상품정보 I/F 누락 사례 (ITM.TITEM 속성 미변경)                      │
│  • 안산 거점(로지스밸리) 구축 시 WMS 연동 변경 범위                        │
│  • 다회용 포장재 운영 시 WMS 프로세스 변경                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 FU → COU (배송지시정보 전송)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                   배송관리(FU) → 택배사연동(COU) 흐름                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  택배사 직송(비센터) 또는 센터 출고 후 택배 인계                           │
│                                                                             │
│  ┌──────────────┐                        ┌──────────────┐                  │
│  │  배송관리    │                        │  택배사연동  │                  │
│  │   (FU)       │                        │   (COU)      │                  │
│  └──────┬───────┘                        └──────┬───────┘                  │
│         │                                        │                          │
│         │ 🟧 배송지시정보가 전송되었다            │                          │
│         │   - 택배사코드, 배송지, 상품정보        │                          │
│         │────────────────────────────────────→   │                          │
│         │                                        │ 택배사별 I/F 방식 분기   │
│         │                                        │                          │
│         │                                        │ 🟧 운송장이 발급되었다   │
│         │                                        │                          │
│         │    🟧 택배사 접수가 확인되었다           │                          │
│         │   ←────────────────────────────────────│                          │
│                                                                             │
│  택배사별 분기:                                                             │
│  • CJ대한통운 (GLS): 직접 API 연동                                        │
│  • 한진/롯데/우체국: EAI 기반 연동                                         │
│  • 로젠/일양로지스: 별도 I/F 방식                                          │
│  • CVSnet/BGFpost: 편의점 배송 전용                                        │
│                                                                             │
│  💜 정책:                                                                   │
│  • "지역별 택배사 이원화 (특정 지역은 지정 택배사)"                        │
│  • "택배사별 I/F 규격에 맞춘 데이터 변환"                                  │
│                                                                             │
│  🩷 핫스팟:                                                                 │
│  • 택배사별 I/F 장애 시 대체 택배사 전환 정책                              │
│  • 택배사 추가 시 COU 서비스 확장 범위                                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.4 COU → TRK (배송정보 수신)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                   택배사연동(COU) → 배송추적(TRK) 흐름                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  택배사로부터 배송 상태 정보를 수신하여 추적 서비스로 전달                  │
│                                                                             │
│  ┌──────────────┐                        ┌──────────────┐                  │
│  │  택배사연동  │                        │  배송추적    │                  │
│  │   (COU)      │                        │   (TRK)      │                  │
│  └──────┬───────┘                        └──────┬───────┘                  │
│         │                                        │                          │
│         │ 🟧 배송정보가 수신되었다                │                          │
│         │   - 운송장번호, 배송상태, 시간           │                          │
│         │────────────────────────────────────→   │                          │
│         │                                        │ 🟧 배송상태가            │
│         │                                        │    변경되었다            │
│         │                                        │  (집화→간선→배달출발     │
│         │                                        │   →배달완료)             │
│         │                                        │                          │
│         │                                        │ 💜 출고SMS 자동 전송     │
│         │                                        │                          │
│         │                                        │ 🟧 배송이 완료되었다     │
│         │                                        │────────────────────→    │
│         │                                        │   🟩 기간계(OR)          │
│         │                                        │   배송상태 업데이트      │
│                                                                             │
│  상태 변경 흐름:                                                            │
│  집화 → 간선상차 → 간선하차 → 배달출발 → 배달완료                         │
│                                                                             │
│  💜 정책:                                                                   │
│  • "출고 시 SMS 자동 발송 (운송장번호 포함)"                               │
│  • "가출고 상태 보류처리 (실제 출고 확인 전)"                              │
│                                                                             │
│  🩷 핫스팟:                                                                 │
│  • 배송상태 변경 빈도가 높을 때 이벤트 발행 전략                           │
│  • 미자(미배달) 정보 처리 프로세스                                         │
│  • 가출고 관리 — 실제 출고와 시스템 출고 시점 불일치                       │
```

## 3. 바운디드 컨텍스트 후보 확인

### 3.1 6개 서비스 = 6개 컨텍스트 검증

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                 6개 MSA 후보 = 6개 바운디드 컨텍스트 검증                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  검증 기준: 각 서비스가 독립된 바운디드 컨텍스트인지 확인                    │
│                                                                             │
│  서비스       │ 독립배포 │ 독립DB │ 독립팀  │ 프로세스축│ 컨텍스트 ✓       │
│  ─────────────────────────────────────────────────────────────────────────  │
│  배송관리(FU) │  ✅      │  ✅    │ ✅     │ 출하~배송 │ ✅ 핵심 허브     │
│  창고관리(WMS)│  ✅      │  ✅    │ ✅     │ 입고~출고 │ ✅ 센터 운영     │
│  택배사(COU)  │  ✅      │  ✅    │ ✅     │ 배송지시  │ ✅ 택배사 연동   │
│  반품/회수    │  ✅      │  ✅    │ ✅     │ 반품~환불 │ ✅ 역물류        │
│  (RTN)        │          │        │         │           │                   │
│  배송추적(TRK)│  ✅      │  ✅    │ 조건부 │ 출고~인수 │ ✅ 상태 추적     │
│  물류정산(STL)│  ✅      │  ✅    │ 조건부 │ 정산      │ ✅ 비용 정산     │
│                                                                             │
│  ⚠️ 주의 사항:                                                              │
│  • FU(배송관리)는 전체 물류의 "허브" 역할 — 다른 서비스 의존도 높음        │
│  • WMS는 외부 시스템(WMS 솔루션) 의존성이 높아 ACL 필수                    │
│  • COU는 택배사별로 I/F가 다르므로 어댑터 패턴 적용 필요                   │
│  • TRK/STL은 팀 독립성이 조건부 — 전담팀 구성 여부에 따라 결정             │
│  • RTN은 기간계(OR)와 경계가 중요 — 반품접수는 OR, 회수처리는 RTN          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
