# MSA 전환 프로젝트 - 이벤트 스토밍 워크샵 가이드

> msa 관련 자료를 모아놓은 문서 repository

## 프로젝트 소개

이 저장소는 **모놀리식 시스템을 마이크로서비스 아키텍처(MSA)로 전환**하기 위한 첫 단계인 **도메인 이벤트 스토밍 워크샵**을 계획하고 실행하기 위한 완전한 가이드를 제공합니다.

### 대상

- MSA 전환 프로젝트를 진행하는 PM
- 이벤트 스토밍 워크샵을 처음 진행하는 퍼실리테이터
- 도메인 경계 식별이 필요한 개발 조직

## ✅ 성공 기준

워크샵이 성공했다면:

- ✓ 주요 비즈니스 프로세스가 이벤트로 시각화됨
- ✓ 참석자들이 도메인 전체 흐름을 이해함
- ✓ 3개 이상의 바운디드 컨텍스트가 식별됨
- ✓ 부서 간 연동 지점이 명확히 파악됨
- ✓ MSA 전환을 위한 구체적인 다음 단계가 도출됨

---

## 🚀 워크샵 이후

### 즉시 (당일)
- 화이트보드 전체 사진 촬영
- 참석자 감사 메일 발송

### 1주일 이내
- 결과물 디지털화 (Miro, Mural 등)
- 전체 공유

### 2주일 이내
- 바운디드 컨텍스트 상세 정의
- 마이크로서비스 후보 도출

### 1개월 이내
- 검증 워크샵 (필요시)
- MSA 전환 로드맵 수립
