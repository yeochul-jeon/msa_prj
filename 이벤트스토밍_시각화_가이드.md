## 1. 전체 프로세스 흐름도

```mermaid

        P1_1["- 참석자"]
        P1_2["- 준비물"]
        P1_3["- 공간"]
    end

    subgraph Phase2[" Phase 2: 이벤트 도출 "]
        P2_1["개별작업:<br/>이벤트 식별"]

    end

    subgraph Phase3[" Phase 3: 정렬 "]

        P3_2["🟧→🟧→🟧→🟧"]
    end

    subgraph Phase4[" Phase 4: 요소 추가 "]
        P4_1["+ 커맨드"]

        P4_4["+ 정책"]

    Phase2 --> Phase4
    Phase4 --> Phase3
```

## 2. 포스트잇 색상별 사용법

### 전체 컬러 팔레트
```
┌─────────────────────────────────────────────────────────────┐
│                       포스트잇 색상 가이드                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  🟧 오렌지    도메인 이벤트                                │
│             "~되었다" (과거형)                             │
│             예: "주문이 접수되었다"                        │
│                                                             │
│  🟦 파란색    커맨드 (명령)                                │
│             "~하기" (명령형)                               │
│             예: "주문하기"                                 │
│                                                             │
│  🟨 노란색    애그리게잇 (데이터 모델)                     │
│             명사형                                         │
│             예: "주문서", "상품"                           │
│                                                             │
│  🟪 연보라    정책/비즈니스 규칙                           │
│             "만약 ~하면"                                   │
│             예: "재고<10이면 알림"                         │
│                                                             │
│  🟥 빨간색    핫스팟 (문제/이슈)                           │
│             개선 필요 영역                                 │
│             예: "배송 지연 빈발"                           │
│                                                             │
│  🟩 연두색    외부 시스템                                  │
│             타 시스템 연동                                 │
│             예: "PG사 API"                                 │
│                                                             │
│  ⬜ 큰 포스트잇  바운디드 컨텍스트                         │
│             도메인 경계                                    │
│             예: "주문 관리"                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 3. 이벤트 스토밍 기본 패턴

### Pattern 1: 기본 흐름 (이벤트 → 이벤트)

```mermaid
flowchart LR
    E1["회원가입되었다"]:::event --> E2["로그인되었다"]:::event
    E2 --> E3["상품검색되었다"]:::event
    E3 --> E4["장바구니 담겼다"]:::event

    classDef event fill:#FF9800,color:#000,stroke:#E65100
```

<details>
<summary>📝 원본 텍스트 다이어그램</summary>

```
   시간 흐름 →

🟧          🟧          🟧          🟧
회원가입    →  로그인    →  상품검색  →  장바구니
되었다        되었다       되었다      담겼다
```

</details>

```
   (고객)        접수되었다
```

</details>

### Pattern 3: 애그리게잇 포함

```mermaid

    C1["결제하기<br/>(고객)"]:::command --> A1["주문서"]:::aggregate

    classDef command fill:#2196F3,color:#fff,stroke:#1565C0
    classDef aggregate fill:#FFEB3B,color:#000,stroke:#F9A825
    classDef event fill:#FF9800,color:#000,stroke:#E65100
```

</details>

## 5. 바운디드 컨텍스트 시각화

### 컨텍스트 맵 예시

```mermaid
flowchart TB
    subgraph Auth["회원/인증 컨텍스트<br/>담당: 보안팀"]
        A1["🟧 회원가입됨"]
        A2["🟧 로그인됨"]
        A3["🟧 권한부여됨"]
    end

    subgraph Product["상품 카탈로그<br/>담당: 상품팀"]
        P1["🟧 상품등록됨"]
        P2["🟧 상품조회됨"]
        P3["🟧 재고변경됨"]
    end

    subgraph Order["주문 관리 컨텍스트<br/>담당: 주문팀"]
        O1["🟧 주문접수됨"]
        O2["🟧 주문승인됨"]
        O3["🟧 주문취소됨"]
        O4["🟧 주문완료됨"]
        OH["🟥 핫스팟: 주문 취소 시<br/>재고 복구 로직 불명확"]
    end

    subgraph Delivery["배송 컨텍스트<br/>담당: 물류팀"]
        D1["🟧 배송시작됨"]
        D2["🟧 배송완료됨"]
        D3["🟧 반품접수됨"]
        DX["🟩 물류사 API"]
    end

    subgraph Payment["결제 컨텍스트<br/>담당: 결제팀"]
        PM1["🟧 결제요청됨"]
        PM2["🟧 결제완료됨"]
        PM3["🟧 환불처리됨"]
        PMX["🟩 PG사 API"]
    end

    Auth -->|"인증"| Product
    Auth -->|"회원정보"| Order
    Product -->|"상품정보"| Order
    Order -->|"주문정보"| Delivery
    Order -->|"결제요청"| Payment
    Payment -->|"환불"| Delivery

    style Auth fill:#E3F2FD,stroke:#1976D2
    style Product fill:#FFF3E0,stroke:#FF9800
    style Order fill:#FCE4EC,stroke:#E91E63
    style Delivery fill:#E8F5E9,stroke:#4CAF50
    style Payment fill:#F3E5F5,stroke:#9C27B0
```
