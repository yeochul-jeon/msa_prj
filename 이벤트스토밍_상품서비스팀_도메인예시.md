│  [출력 - 판매자에게 보내는 것]                             │
│  ┌──────────────────┬──────────────────────────────────┐  │
│  │ 이벤트           │ 데이터 항목                       │  │

│  │ 상품 등록 승인   │ 상품ID, 승인일시                 │  │
│  │ 상품 등록 반려   │ 상품ID, 반려사유, 수정가이드     │  │

│  │ 가격 변경 반려   │ 상품ID, 반려사유                 │  │
│  │ 상품이 품절됨    │ 상품ID, 품절일시                 │  │

│  └──────────────────┴──────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

### 6.3 이벤트 소유권 원칙

```
┌─────────────────────────────────────────────────────────────┐
│                  이벤트 소유권 원칙                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  핵심 원칙: "누가 데이터를 관리하는가?"                    │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  데이터 관리 주체 = 이벤트 발행 주체                       │
│                                                             │
│  예시:                                                     │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ 데이터                 │ 소유 팀    │ 이벤트 발행 팀 │  │
│  ├──────────────────────────────────────────────────────┤  │
│  │ 상품 기본 정보         │ 상품팀     │ 상품팀         │  │
│  │ 상품 재고              │ 상품팀     │ 상품팀         │  │
│  │ 상품 가격 (정가/판매가)│ 상품팀     │ 상품팀         │  │
│  │ 할인가 (프로모션)      │ 프로모션팀 │ 프로모션팀     │  │
│  │ 검색 색인              │ 검색팀     │ 검색팀         │  │
│  │ 입출고 정보            │ 물류팀     │ 물류팀         │  │
│  │ 주문 정보              │ 주문팀     │ 주문팀         │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  상품팀의 역할:                                            │
│  • 상품/재고/가격 관련 이벤트 "발행"                      │
│  • 다른 팀의 요청을 "수신"하여 처리 후 결과 발행          │
│                                                             │
│  예시:                                                     │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  🟩 주문팀                                           │  │
│  │     │                                                 │  │
│  │     │ "재고 예약 요청" (주문팀 발행)                 │  │
│  │     │                                                 │  │
│  │     ▼                                                 │  │
│  │  상품팀 (수신 → 처리)                                │  │
│  │     │                                                 │  │
│  │     │ 재고 확인 및 예약 처리                         │  │
│  │     │                                                 │  │
│  │     ▼                                                 │  │
│  │  "재고가 예약되었다" (상품팀 발행)                   │  │
│  │     │                                                 │  │
│  │     └──→ 🟩 주문팀 (수신)                           │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.4 연동 이벤트 명명 규칙

```
┌─────────────────────────────────────────────────────────────┐
│                  연동 이벤트 명명 규칙                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 요청 이벤트 (타팀 → 상품팀)                            │
│     • "~가 요청되었다"                                     │
│     • 예: "재고 예약이 요청되었다"                         │
│     • 예: "색인이 요청되었다"                              │
│                                                             │
│  2. 결과 이벤트 (상품팀 → 타팀)                            │
│     • "~가 완료되었다" / "~가 실패했다"                    │
│     • 예: "재고가 예약되었다", "재고 예약이 실패했다"      │
│                                                             │
│  3. 상품 상태 이벤트 (상품팀 발행)                         │
│     • "상품 ~가 ~되었다"                                   │
│     • 예: "상품이 등록되었다", "상품이 품절되었다"         │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  구분 예시:                                                │
│                                                             │
│  ❌ 혼란스러운 명명:                                       │
│  • "재고 차감됨" → 상품팀? 주문팀? 물류팀?                │
│                                                             │
│  ✅ 명확한 명명:                                           │
│  • 주문팀: "재고 차감이 요청되었다" (요청)                │
│  • 상품팀: "재고가 차감되었다" (결과, 상품팀 발행)        │
│  • 물류팀: "출고가 완료되었다" (물류팀 발행)              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
