## 2. 주문 상태 이벤트 vs 내부 처리 구분

### 2.1 핵심 판단 기준

```
┌─────────────────────────────────────────────────────────────┐
│                    핵심 판단 기준                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  "이 변화로 다른 팀이 행동을 취해야 하는가?"               │
│                                                             │
│  예 → 도메인 이벤트 가능성 높음 🟧                         │
│  아니오 → 내부 처리 단계 (제외)                            │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  추가 판단 기준:                                           │
│  • 고객에게 알림이 발송되는가?                             │
│  • 주문 상태가 고객에게 보여지는가?                        │
│  • 이 상태에서 고객이 행동을 취할 수 있는가?               │
│  • 다른 시스템이 이 이벤트를 구독하는가?                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 판단 체크리스트 (3단계)

이벤트 후보가 도메인 이벤트인지 판단할 때 아래 질문을 순서대로 확인하세요.

```
□ 1. 주문 상태가 변경되는가?
     → 아니오: 내부 처리 (제외)
     → 예: 다음 질문으로

□ 2. 고객이 이 변화를 인지할 수 있는가?
     → 아니오: 내부 처리 가능성 높음
     → 예: 다음 질문으로

□ 3. 다른 팀/시스템이 이 변화에 반응해야 하는가?
     → 아니오: 🩷 핫스팟으로 표시 (추가 논의)
     → 예: 도메인 이벤트! 🟧
```
